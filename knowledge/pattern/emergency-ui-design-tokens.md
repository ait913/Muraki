## Context

「人が倒れた」「火災が起きた」など極度のパニック時にユーザーが操作する UI の設計。
通常の UX 原則 (情報密度・装飾・アニメ多用) は **逆効果**。アドレナリン下では
- 視野狭窄 (cognitive tunneling) で1要素しか見えない
- 文章は読めない (フレーズしか入らない)
- 指先精度が落ちる (誤タップ多発)
- 「戻る」「キャンセル」を反射で押す

ため、通常のデザインシステムから外れた専用 token と挙動原則が要る。

## What

実証済みの設計トークン:

| 項目 | 値 |
|---|---|
| 主要素の画面占有率 | **60%以上** (1画面1主要素) |
| 文字数上限 | **12文字** (主要素のラベル) |
| フォントサイズ最小 | **24pt** (補助情報も)。主要素は 40pt+ |
| コントラスト比 | **7:1 (WCAG AAA)**。dark 固定 (黒地+白文字+赤アクセント) |
| タップ領域 | **88pt 以上** (iOS デフォの2倍) |
| 1画面の選択肢 | **1〜3、最大4** (Hick's Law) |
| キャンセル/戻る | **隠す or 長押し2秒** (誤タップ防止、緊急画面に「戻る」を置かない) |
| 進捗表示 | 常時表示 (「あと2分10秒」) で不確実性を消す |
| 撤退/中断 | 罪悪感ゼロ表現 (「対応できなくて大丈夫です」) |
| 音声優先 | 視覚は確認のみ、誘導は声で (両手が塞がる前提) |

挙動原則:
- カウントダウンキャンセルは **長押し2秒**、1秒以下のタップでは反応しない
- 巨大 SOS ボタン (直径 280pt+) を画面中央に固定
- ライトモード非対応 (眩しさ低減・コントラスト保証のため dark 固定)
- 待機時の脈動アニメ (scale 1.0→1.03、5秒周期) で「生きている」感
- Haptics: 重要操作は `Heavy`、確定は `Success`、注意は `Warning`

## Why

- **Cognitive Tunneling**: アドレナリン下では1要素しか視認できない。情報密度を下げないと「何を押せばいいか」が分からない。
- **指先精度低下**: 心拍上昇でタップ精度が落ちる。88pt 以上ないと誤タップが頻発する。
- **読解力低下**: 文章を読む脳のリソースが奪われる。フレーズ (動詞+助詞 程度) しか入らない。
- **反射的な誤キャンセル**: 緊急時に「戻る」を反射で押して全部やり直しになるのを防ぐため、キャンセルは隠す。
- **罪悪感ゼロ設計**: 撤退できる選択肢がないと、無理な救助で二次被害 → ユーザーが二度と使わなくなる。

## How to apply

新規緊急アプリ・防災 UI を設計する時は:

1. **デザインシステムをコピペで開始**: `colors.ts` / `typography.ts` を上記値で初期化
2. **画面ごとに「主要素1つ」を決める**: 残りは副情報、画面の60%を主要素に割く
3. **タップ領域チェックリスト**: 全 Pressable に `hitSlop` 含めて 88pt 以上を確保
4. **キャンセル動線監査**: 緊急画面に `headerLeft` の戻るボタンを置かない、長押し前提
5. **アクセシビリティ**: VoiceOver/TalkBack で操作可能を必須要件にする (主要素には `accessibilityLabel`、進捗には `accessibilityLiveRegion="polite"`)
6. **dark 固定**: ライトモードを諦めることで眩しさ・コントラスト両方を保証
7. **音声 + Haptics の併用**: 視覚に依存しない誘導を音声 + 振動で構築
8. **罪悪感ゼロのコピー**: 「無理」「諦める」を選んだ後は「対応できなくて大丈夫です」と言い切る
