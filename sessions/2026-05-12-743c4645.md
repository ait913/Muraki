## 課題点

- **Codex 起動が trusted dir 制約で動かなかった**
  - **何が課題か**: Researcher 規約は「特定 API 確認 = Codex 併用」だが、`codex exec` が trusted directory 設定外で起動できず、Gemini + WebFetch + Bash の組み合わせで代替した
  - **いつ起こるか**: Researcher 召集時、特に worktree 外や `/tmp` 経由で Codex 起動を試みた場合
  - **次どうするか**: `knowledge/tool-quirk/codex-exec-sandbox-default.md` に「trusted dir 設定 / 回避手順」を追記するか、Researcher の規約自体を「Codex が動かない時は Gemini + Bash でも可」と明文化

- **「セッションレポート」要望を最初にタスク管理と誤読した**
  - **何が課題か**: 1 ターン目で「TODO/予定」と「事実ベース振り返り」の区別を確認しなかったため、push back の方向が誤り、Touri にもう一度説明させた
  - **いつ起こるか**: 抽象度の高い提案を受けた時。「report」「ログ」「記録」のような単語は意味が広い
  - **次どうするか**: 抽象語が出た時は「TODO 系か / 事実ベース系か」「prospective か / retrospective か」を最初に確認する。AskUserQuestion で 1 問だけ確認するコストの方が安い

- **INDEX gen の blockquote 拾いバグが放置されていた**
  - **何が課題か**: `first_para()` が `>` で始まる行も summary として拾うため、ファイル冒頭の注意書き blockquote が INDEX のサマリに出ていた
  - **いつ起こるか**: knowledge ファイルの冒頭に注意書き / pointer を `>` で書いた時
  - **次どうするか**: 今回 1 行修正済 (`s.startswith(">")` で skip)。同種の表面化していないバグが他にないか、`scripts/` の他ファイルもいずれ眺める

  - **何が良かったか**: 既存 knowledge が今でも正しいか裏取りでき、補正 3 点 + 追記 4 点が確定。「整形 = 表記揃え」で済ませず一次情報照合まで踏めた
